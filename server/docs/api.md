# Server API

The public API is owned by FastAPI and remains independent of the selected LLM
provider. JSON is encoded as UTF-8.

Annotated examples use JSONC for readability. Remove `//` comments before
sending them as JSON request bodies.

## Health Check

```http
GET /ping
```

```json
{
  "status": "ok",
  "service": "avatar-agent-server"
}
```

`status: string` reports service health, while `service: string` is the stable
backend identifier.

## Chat

```http
POST /api/v1/unity/chat
Content-Type: application/json
```

Request:

```jsonc
{
  "session_id": "unity-demo", // string, conversation identifier (1-128 chars)
  "message": "Today we completed the first integration." // string, user input
}
```

Both values must be non-blank. `message` accepts up to 10,000 characters.

Response:

```jsonc
{
  "request_id": "req_3f887b84e5304fc6b52a7be10ba34b45", // string, log correlation ID
  "reply": "Echo: Today we completed the first integration.",
  "emotion": {
    "label": "neutral", // enum, dominant emotion
    "valence": 0.0, // float [-1, 1], negative to positive direction
    "arousal": 0.0, // float [0, 1], calm to highly activated
    "intensity": 0.0 // float [0, 1], emotion strength
  },
  "avatar": {
    "expression": "neutral", // enum, VRM expression requested from Unity
    "action": "idle", // enum: idle, nod, or wave
    "intensity": 0.0, // float [0, 1], expression/action blend weight
    "duration_ms": 0 // integer [0, 60000], 0 uses the client default
  }
}
```

`valence` describes whether an emotion is negative or positive; `arousal`
describes its energy level independently. Supported emotion/expression labels
are `neutral`, `happy`, `sad`, `angry`, `relaxed`, and `surprised`.

The reply is generated by the active LLM provider. The default provider is
`mock`, which preserves the deterministic `Echo: ...` behavior without making
an external request. The emotion and avatar fields remain neutral until their
modules are connected.

Provider configuration and LLM failures use these status codes:

- `400`: the runtime provider configuration is incomplete or unsafe
- `502`: the provider rejected the request or returned an invalid response
- `504`: the provider request exceeded the configured timeout

Invalid or missing fields return HTTP 422:

```json
{
  "detail": [
    {
      "type": "value_error",
      "loc": ["body", "message"],
      "msg": "Value error, must not be blank",
      "input": "   "
    }
  ]
}
```

## Runtime LLM Configuration

Unity and web clients share one process-wide runtime configuration endpoint:

### Provider catalog

```http
GET /api/v1/llm/providers
```

The catalog returns curated provider/model choices for client menus. Each
model's `config` object can be posted directly to the configuration endpoint:

```jsonc
{
  "providers": [
    {
      "provider": "openai",
      "display_name": "OpenAI",
      "default_model": "gpt-5.6-luna", // string, server-recommended default
      "allow_custom_model": true, // boolean, accepts models outside this list
      "models": [
        {
          "model": "gpt-5.6-luna",
          "display_name": "GPT-5.6 Luna",
          "config": { // object, may be posted directly to /api/v1/llm/config
            "provider": "openai",
            "model": "gpt-5.6-luna"
          }
        }
      ]
    }
  ]
}
```

The catalog never contains API keys or base URLs. Advanced clients may still
send custom `model`, `base_url`, and `api_key` values directly. Unity only needs
the display names for menus and the nested `config` payload for configuration.

### Configure the runtime

```http
POST /api/v1/llm/config
Content-Type: application/json
```

The last successful request becomes active for all clients. The configuration
is kept only in memory and is reset when the server restarts.

Supported providers and defaults:

| Provider | Default model | Default base URL | Environment key |
| --- | --- | --- | --- |
| `openai` | `gpt-5.6-luna` | `https://api.openai.com/v1` | `OPENAI_API_KEY` |
| `gemini` | `gemini-3.5-flash` | `https://generativelanguage.googleapis.com/v1beta` | `GOOGLE_API_KEY` or `GEMINI_API_KEY` |
| `deepseek` | `deepseek-v4-flash` | `https://api.deepseek.com` | `DEEPSEEK_API_KEY` |
| `mock` | `mock-echo` | local only | none |

With the provider API key set in the server environment, only the provider is
required:

```bash
curl -X POST http://127.0.0.1:8000/api/v1/llm/config \
  -H "Content-Type: application/json" \
  -d '{"provider":"openai"}'
```

PowerShell must preserve the JSON quotes when calling `curl.exe`:

```powershell
curl.exe -X POST http://127.0.0.1:8000/api/v1/llm/config `
  -H "Content-Type: application/json" `
  --data-raw '{\"provider\":\"openai\"}'
```

For local development, a client can provide the key and optional overrides in
the same request body:

```jsonc
{
  "provider": "deepseek", // enum, required provider identifier
  "api_key": "replace-with-local-key", // string or null, request-only secret
  "model": "deepseek-v4-flash", // string or null, defaults to server config
  "base_url": "https://api.deepseek.com" // HTTP(S) URL or null
}
```

Only `provider` is required when its API key exists in the server environment.
A custom `base_url` requires `api_key` in the same request.

Response:

```jsonc
{
  "provider": "deepseek",
  "model": "deepseek-v4-flash",
  "base_url": "https://api.deepseek.com", // string, resolved adapter endpoint
  "api_key_configured": true, // boolean, key is usable but never returned
  "api_key_source": "request" // enum: request, environment, or not_required
}
```

The response summarizes the active runtime configuration without exposing the
credential itself.

API keys are never returned, persisted, or logged. To prevent a server
environment key from being forwarded to an arbitrary host, a custom `base_url`
is accepted only when `api_key` is supplied in the same request.

This unauthenticated configuration endpoint is intended only for the local
demo server bound to `127.0.0.1`. Add authentication and TLS before exposing
them to an untrusted network.
